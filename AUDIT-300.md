# Pinpoint 300-Org Security Audit — Target List

## Selection Criteria

1. **Active:** Committed to a public repo in the last 90 days
2. **Uses Actions:** Has repos with .github/workflows/
3. **Scale:** 10+ public repos minimum
4. **Relevance:** Industry leaders whose practices set the standard
5. **Public only:** Only auditing public repos (no auth needed)

## Audit Process

### For each org:
1. `pinpoint audit --org <name> --output json > audits/<name>.json`
2. Capture: total repos, total workflows, total action refs, % SHA-pinned,
   % tag-pinned, % branch-pinned, Docker actions count, transitive dep count,
   known advisory matches, GPG signing status
3. Filter out repos with zero workflows (no Actions usage)
4. Filter out repos with no commits in 90 days (dead)
5. Record timestamp of audit

### Aggregation:
- Per-org scorecard (A-F grade based on pinning %)
- Per-industry rollup
- Top 20 most-referenced unpinned actions across all 300 orgs
- Top 10 actions with advisory matches
- Docker action usage stats (how many orgs use Docker-type actions)
- Transitive dependency depth distribution

### Report Outputs:
- Full data: JSON per org (private, for our analysis)
- Public report: aggregated stats, per-industry breakdown, named org scorecards
- Blog post: narrative with key findings
- SARIF files: can offer to any org that wants their detailed results
